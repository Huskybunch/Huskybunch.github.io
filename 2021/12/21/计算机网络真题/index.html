Internet协议栈中的五层是什么?每一层的主要职责是什么?

The following are the five layers in the Internet protocol stack:

  1. Application layer
  2. Transport layer
  3. Network layer
  4. Data link layer
  5. Physical layer.

Principal responsibilities:

Application layer: HTTP, SMTP, and FTP protocols are used in application layer. it is used to send data over multiple end systems.

**应用层:**应用层采用HTTP、SMTP、FTP等协议。它用于在多个终端系统上发送数据。

Transport layer: TCP and UDP protocols are used in transport layer. Transfer the content between two endpoints mainly.

**传输层:**主要传输两个端点之间的内容。传输层采用TCP和UDP协议。

Network layer: IP protocol is used in network layer. Move the packets between any two hosts in the network.

**网络层:**在网络中的任意两个主机之间移动数据包。网络层采用IP协议。

Data link layer: Point-to-point protocol (ppp) used in data link layer.Move the packets from one node to the next another node.

**数据链路层:**将数据包从一个节点移动到另一个节点。数据链路层使用的点到点协议(ppp)。

Physical layer: Transfer the individual bits from one node to the next node with in the frame.

物理层:将单个比特从一个节点传输到帧中的下一个节点。

  1. Decreases the delay the user experiences before and during a call

    减少用户在通话前和通话中所经历的延迟

  2. The call will be done with a steady bandwidth, dedicated channel, and consistent data rate

    通话将在稳定的带宽、专用信道和一致的数据速率下完成

  3. Packets are always delivered in the correct order

    数据包总是按照正确的顺序发送

  • Doesn’t use resources efficiently
  • Dedicated channels for circuit switching are unavailable for any other use
  • There is a higher cost to dedicate one channel per use

Advantages of packet switching over circuit switching:

  • More efficient than circuit switching
  • 比电路开关更高效
  • Data packets are able to find the destination without the use of a dedicated channel
  • 数据包能够在不使用专用通道的情况下找到目的地
  • Reduces lost data packets because packet switching allows for resending of packets
  • 减少数据包的丢失，因为数据包交换允许重发数据包
  • More cost-effective since there is no need for a dedicated channel for voice or data traffic
  • 更节省成本，因为不需要一个专用通道的语音或数据流量

why FTP out of band

FTP uses two parallel TCP connections, one connection for sending control information (such as a request to transfer a file) and another connection for actually transferring the file. Because the control information is not sent over the same connection that the file is sent over, FTP sends control information out of band.

FTP使用两个并行的TCP连接，一个连接用于发送控制信息(例如传输文件的请求)，另一个连接用于实际传输文件。因为控制信息不是通过发送文件的同一连接发送的，所以FTP将控制信息发送到带外。

ARP查询以广播帧的形式发送，因为查询的主机不知道哪个适配器地址对应该IP地址。对于响应，发送节点知道应向其发送响应的适配器地址，因此不需要发送广播帧(这必须由局域网中的所有其他节点处理)。

Suppose that Alice wants to send an email message to Bob. This will involve four entities: Alice’s mail client (for email composition and sending), Alice’s outgoing mail server, Bob’s incoming mail server, and Bob’s mail client (for email retrieval and viewing). Between which of these four entities does the SMTP protocol operate? What about the IMAP protocol?

假设Alice想要向Bob发送一封电子邮件。这将涉及四个实体:Alice的邮件客户端(用于电子邮件的组合和发送)、Alice的发送邮件服务器、Bob的接收邮件服务器和Bob的邮件客户端(用于电子邮件的检索和查看)。SMTP协议在这四个实体中的哪一个之间操作?那么IMAP协议呢?

Answer: SMTP runs between Alices mail client and her server, and also (separately) between her server and Bob’s server. IMAP runs between Bob’s server and his mail client to retrieve messages from Bob’s server.

答:SMTP在alice邮件客户端和她的服务器之间运行，也在她的服务器和Bob的服务器之间(单独地)运行。IMAP在Bob之间运行服务器和他的邮件客户机来从Bob的服务器检索消息。
