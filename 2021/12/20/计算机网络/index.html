What is computer networks

A computer network is a set of nodes that are connected together by communication links.

计算机网络是由通信链路连接在一起的一组节点。

  • Set of nodes

    Computers, switches, routers, hubs, printers, and servers, smart phone, tablet

    电脑、交换机、路由器、集线器、打印机、服务器、智能手机、平板电脑

  • Communication links

    Fiber optics, Coaxial pair,cable, copper, radio wave, micro wave

    光纤、同轴对、电缆、铜、无线电波、微波

Server

A server is a computer or system that provides resources, data,services, or programs to other computers, known as clients, over anetwork. Types: File server, Domain Name System(DNS) server, webserver, print server, mail server etc.

服务器是通过网络向其他计算机(即客户端)提供资源、数据、服务或程序的计算机或系统。类型: 文件服务器，域名系统(DNS)服务器，网络服务器，打印服务器，邮件服务器等

Client

a client is a computer that accesses a service made available by a server.

客户端是访问由服务器提供的服务的计算机。

Hub

Hub is a device that splits a network connection into multiple computers.

集线器是一种将网络连接分割成多台计算机的设备。

Router

A Router is a networking device that receives and forwards data packets between computer network.

路由器是一种网络设备，在计算机网络之间接收和转发数据包。

Data cable

Data cables are used to transmit the information from a source to a destination.

数据电缆用于将信息从一个源传输到一个目的地。

Peers

a peer is a computer that accessesa service made available by a server.

对等点是一台可以访问服务器提供的服务的计算机。

Components of data communication system in computer networks计算机网络中数据通信系统的组成部分

Sender 发报机

It is the device which sends the data messages. It can be a computer, workstation, telephone handset

它是发送数据消息的设备。它可以是电脑、工作站、电话筒

Receiver 接收

It is the device which receives the data messages. It can be a computer, workstation, telephone handset.

Message

It is the information to be communicated. Popular forms ofinformation include text, numbers, images, audio, video etc.

它是要传达的信息。常用的信息形式包括文本、数字、图像、音频、视频等。

Transmission Medium传送介质

It is the physical path by which a message travels from sender to receiver. Some examples include twisted pair wire, coaxial cable, radio waves etc.

它是消息从发送方传递到接收方的物理路径。例如双绞线、同轴电缆、无线电波等。

Protocol 协议

It is a set of rules that governs the data communications. It represents an agreement between the communicating devices. Without a protocol, two devices may be connected but not communicating.

它是一组管理数据通信的规则。它代表通信设备之间的协议。如果没有协议，两个设备可以连接但不能通信。

Transmission modes in computer networks计算机网络中的传输方式

Transmission mode means transferring of data between two devices. It is also knownas communication mode. 传输模式是指在两台设备之间传输数据。它也被称为通信模式。

Simplex Mode单工方式

  • In Simplex mode, the communication is unidirectional, as on a one-way street.

    在单工模式下，通信是单向的，就像在一条单行道上一样。

  • Only one of the two devices on a link can transmit, the other can only receive.

    链路上的两台设备中只有一台可以发送，另一台只能接收。

  • The simplex mode can use the entire capacity of the channel to send data in one direction.

    单工模式可以使用通道的全部容量在一个方向上发送数据。

例子:键盘和传统显示器。键盘只能引入输入，显示器只能给出输出。

Half-Duplex Mode半双工

  • In half-duplex mode, each station can both transmit and receive, but not at the same time.

    在半双工模式下，每个站都可以发送和接收，但不能同时进行。

  • When one device is sending, the other can only receive, and vice versa. 当一个设备发送时，另一个只能接收，反之亦然。

  • The half-duplex mode is used in cases where there is no need for communication in both directions at the same time.

    半双工模式用于不需要同时在两个方向上进行通信的情况。

  • The entire capacity of the channel can be utilized for each direction.

    通道的全部容量可用于每个方向

例如:一种对讲机，一次设置一个消息，消息在两个方向发送。

Full-Duplex Mode全双工

  • Full-duplex mode is used when communication in both directions is required

    当需要双向通信时，采用全双工模式

  • In full-duplex mode, both stations can transmit and receive simultaneously. In full-duplex mode, signals going in one direction share the capacity of the link with signals going in other direction, this sharing can occur in two ways:

    在全双工模式下，两个站点可以同时发送和接收。在全双工模式下，一个方向的信号与另一个方向的信号共享链路的容量，这种共享可以以两种方式发生:

    • Either the link must contain two physically separate transmissionpaths, one for sending and other for receiving

      任何一条链路都必须包含两条物理上独立的传输路径，一条用于发送，另一条用于接收

    • The capacity is divided between signals travelling in both directions.

      容量被分配到两个方向的信号之间。

例如:电话网络，两者可以同时说话和听。

Data communication ways in computer networks计算机网络中的数据通信方式

Unicast: 单播

  • Unicast is the term used to describe communication where a piece of information is sent from one point to another point. In this case there is just one sender, and one receiver.

    单播是指将一条信息从一个点发送到另一个点的通信。在这种情况下，只有一个发送方和一个接收方。

  • Unicast transmission, in which a packet is sent from a single source to a specified destination.

    单播，即报文从一个源发送到指定的目的地址。

例如:两个人之间的电话。

Broadcast:广播

  • Broadcast is the term used to describe communication where a piece of information is sent from one point to all other points.

    广播是用来描述信息从一个点发送到所有其他点的通信的术语。

  • In this case there is just one sender, but the information is sent to all connected receivers.

    在这种情况下，只有一个发送方，但信息被发送到所有连接的接收方。

例如:电视、广播、广告、网站、在线流媒体和数字新闻

Multicast:组播

  • Multicast is the term used to describe communication where a piece of information is sent from one point to a set of other points.

    组播是指将一条信息从一个点发送到另一组点的通信。

  • In this case, the information is distributed to a set of receivers.

    在这种情况下，信息被分配到一组接收器上。

例子：一个使用多播的应用程序的例子是一个视频服务器发送它的特定的网络电视频道。

Overview of delay

As a packet travels from one node (host or router) to the

subsequent node (host or router) along this path, the packet suffers

from several types of delays at each node along the path.

The most important of these delays are the

- processing delay,

- queuing delay,

- transmission delay

- propagation delay

Together, these delays accumulate to give a total delay

当信息包从一个节点(主机或路由器)沿着这条路径传输到随后的节点(主机或路由器)时，信息包在这条路径上的每个节点都会遭受几种类型的延迟。

这些延迟中最重要的是

——处理延迟,

——排队延迟,

——传输延迟

——传播延迟

这些延迟加在一起就形成了一个总的延迟

Four sources of packet delay

处理延迟:检查数据包的报头和确定数据包的方向所需要的时间是处理延迟的一部分。

dproc:处理延迟

  • 校验位错误

  • 确定输出链接

排队延迟:

在队列中，包在等待传输到链路上时经历排队延迟。特定数据包的排队延迟长度将取决于排队等待传输到链路上的较早到达的数据包的数量。

dqueue:排队延迟

  • 在输出链路等待传输的时间

  • 取决于路由器的拥塞程度

**传输延迟:**这是将数据包位从节点传输到链路输出缓冲区所需的时间量。

传输延迟为L/R。

dtrans :传输延迟

  • L:包长度(位)

  • R:链路带宽(bps)

  • dtrans = L/R

**传播延迟:**一个比特一旦进入链路，就需要传播到路由器B。从链路开始传播到路由器B所需要的时间就是传播延迟。

dprop:传播延迟:

  • d:物理链路长度
  • s:在介质中的传播速度(~2x108米/秒)
  • dprop = d/s

Packet Switching: queueing delay, loss分组交换:排队延迟、丢失

如果到达链路的速率(单位为比特)超过一段时间内链路的传输速率:

  • 数据包将排队，等待在链路上传输如果内存(缓冲区)被填满

  • 数据包会被丢弃(丢失)

Packet loss

由于缓冲区空间是有限的，一个到达的包可能会发现缓冲区被其他等待传输的包填满了

在这种情况下，将发生包丢失——到达的包或已经排队的包之一将被丢弃

Packet switching versus circuit switching分组交换与电路交换

电路交换:

  • 不需要分解成数据包
  • 需要建立连接才能启动传输
  • 它不是一种存储和转发技术

分组交换:

  • 数据被分成若干块，称为数据包

  • 启动传输不需要建立连接

  • 这是一种存储转发技术

Throughput

吞吐量是发送方和接收方之间传输比特的速率(比特/时间单位)

  • 瞬时:给定时间点的速率

  • 平均:时间段的速率

LAN局域网

使用局域网时:

2.1 principles of network applications

TCP/IP

  • “TCP/IP”表示传输控制协议/互联网协议。
  • 它是由国防高级研究计划局创建的(DARPA)。
  • 通信协议的集合。
  • 协议是一组规则，管理网络上的每一种可能的通信。这些协议描述了在internet上源和目的地之间的数据传输。
  • 计算机系统通过网络进行通信有几个层次
  • 应用层为第一层。

Application layer

TCP/IP模型中的应用层是与最终用户最接近的一层。

这意味着应用层和最终用户可以直接与软件应用程序交互。

应用层程序是基于客户端和服务器的。

文件传输协议的类型(FTP)

FTP中有两种类型的连接

  • 控制连接:
  • 通过控制连接，我们可以一次传输命令行或响应行。
  • 数据连接:
  • 数据传输过程之间的数据连接。数据连接在传输文件的命令出现时打开，在传输文件时关闭。

P2P architecture

  • 点对点网络允许对等点之间的通信而不需要服务器。
  • 与客户端-服务器架构不同，P2P架构中没有处理请求的中央服务器。
  • 对等体之间直接交互，不需要中央服务器。

Processes communicating通信进程

当一个程序处于运行状态时称为进程。

  • 运行在多个终端系统中的 程序 通过进程相互通信。
  • 进程可以被看作是一个运行在终端系统中的程序。

通过交换消息相互通信

  • 两个不同端系统上的进程通过计算机网络交换消息进行通信。
  • 发送过程创建并向网络发送消息
  • 接收进程接收这些消息，并可能通过发送消息返回

网络应用程序由成对的进程组成，这些进程通过网络互相发送消息。

例如，在Web应用程序中，客户机浏览器进程与Web服务器进程交换消息

对于P2P,

在P2P文件共享中，下载文件的对等体被标记为客户端，上传文件的对等体被标记为服务器。

Client进程:发起通信的进程。

server进程:等待通信的进程。

Sockets


进程通过socket发送/接收消息

  • 大多数应用程序由成对的通信进程组成，每对进程中的两个进程互相发送消息。
  • 进程通过一个称为套接字的软件接口向网络发送消息，并从网络接收消息。

socket类似于门

  • 在客户端，socket是客户端进程和TCP连接之间的门
  • 在服务器端，它是服务器进程和TCP连接之间的门

Services of transport-layer protocol 传输层协议的服务

Mainly 4 different services that a transport-layer protocol can offer一个传输层协议主要可以提供4种不同的服务

  1. Data integrity

  2. Throughput

  3. Timing

  4. Security

Data integrity数据完整性

一个数据包可能会在计算机网络中丢失。例如，一个包可能会溢出路由器的缓冲区，或者在它的一些位损坏后被主机或路由器丢弃。对于很多应用来说，比如电子邮件、文件传输和金融应用，数据丢失会带来毁灭性的后果

因此，为了支持这些应用程序，必须做一些事情来保证应用程序一端发送的数据是正确和完整的

有些应用程序(例如，文件传输，财务应用)需要100%可靠的数据传输

如果协议提供了这样一种有保证的数据交付服务，就称为提供了可靠的数据传输。

一个传输层协议可以潜在地提供给应用程序的一个重要服务是过程到过程的可靠数据传输。

Throughput吞吐量

在上一章中，我们介绍了可用吞吐量的概念，在网络路径上两个进程之间的通信会话的上下文中，可用吞吐量是指发送进程向接收进程传送比特的速率。

由于在网络路径上共享带宽特性，可用吞吐量可能随时间波动。

一些对带宽敏感的应用程序需要以特定速率保证可用吞吐量。 例如，多媒体

传输层协议可以在特定的速率(r位/秒)下提供有保证的可用吞吐量。

Timing定时

由于传输层协议可以在特定的速率(r位/秒)下提供有保证的可用吞吐量，因此也可以提供定时保证。因为时间与吞吐量相关。

有些应用程序(如网络电话、互动游戏)需要定时保证

传输层协议还可以提供定时保证。

Security

最后，传输协议可以为应用程序提供一个或多个安全服务。

例如，在发送主机中，传输协议可以对发送过程传输的所有数据进行加密，而在接收主机中，传输层协议可以在将数据发送给接收过程之前对数据进行解密。

这种服务将在两个过程之间提供保密性，即使数据在发送和接收过程之间被观察到。

Web and HTTP

web

万维网缩写为WWW，通常被称为Web。

WWW是1989年由欧洲核子研究中心(CERN)发起的。

直到20世纪90年代早期，因特网主要被研究人员、学者和大学生用来登录远程主机，

主要用于本地主机和远程主机之间的文件传输，接收和发送新闻，接收和发送电子邮件。

Web页面(也称为文档)由对象组成。

  • 对象是简单的文件，如HTML文件、JPEG图像或视频剪辑——可通过单个URL寻址。
  • 例如，如果一个网页包含HTML文本和5张JPEG图像，那么Web页面就有6个对象:基本HTML文件加上5个对象图像。
  • 每个URL有两个组件:服务器的主机名存放对象和对象的路径名。

www.someSchool.edu作为主机名和

Components of Web

web有3个组成部分:

  1. URL (Uniform Resource Locator): web上的资源系统。

  2. HTTP (Hyper Text Transfer Protocol):浏览器和服务器之间的通信协议。

  3. 超文本标记语言(Hyper Text Markup Language, HTML):定义网页的结构、组织和内容。

URL (Uniform Resource Locator): web上的资源系统。

URL是Web上给定的唯一资源的地址。

它也被称为网址

HTML代表超文本标记语言。它用于使用标记语言设计网页。HTML是超文本和标记语言的结合。超文本是一种包含链接的文本

一种标记语言是用来在定义网页结构的标签中定义文本文档的。大多数标记语言(如HTML)是人类可读的。该语言使用标记来定义必须对文本进行哪些操作。HTML是一种标记语言，浏览器使用它来操作文本、图像和其他内容，以便以所需的格式显示

http

HTTP(超文本传输协议)定义了Web客户端如何向Web服务器请求Web页面，以及Web服务器如何向客户端传输Web页面。

当用户请求一个Web页面(例如，单击一个超链接)时，浏览器发送页面中对象的HTTP请求消息发送给服务器。服务器接收请求并使用包含对象的HTTP响应消息进行响应。

  • Web的应用层协议

  • 客户机/服务器模型

    • 客户端:请求，接收，(使用HTTP协议),“显示”Web对象
    • 服务器:Web服务器发送(使用HTTP协议)对象响应请求

HTTP使用TCP作为它的底层传输协议

HTTP客户端首先向服务器发起TCP连接(创建套接字)。一旦连接建立，浏览器和服务器通过socket接口访问TCP。并且消息在浏览器(HTTP客户端)和Web服务器(HTTP服务器)

Non-persistent HTTP

1a. HTTP客户端在端口80的www.someSchool.edu发起到HTTP服务器(进程)的TCP连接

1b. 主机www.someSchool.edu上的HTTP服务器正在80端口等待TCP连接。接受连接，通知客户端

  1. HTTP客户端向TCP连接套接字发送HTTP请求消息(包含URL)。消息表明客户端需要对象someDepartment/home.index

  2. HTTP服务器接收请求消息，形成包含请求对象的响应消息，并将消息发送到套接字中

  3. HTTP服务器关闭TCP连接。

  4. http客户端收到响应消息，其中包含html文件，显示html。解析html文件，找到10个引用的jpeg对象

  5. 对10个jpeg对象中的每个对象重复步骤1-5

  • 文件传输时间

SMTP

SMTP是简单邮件传输协议的缩写

它是互联网电子邮件的核心

SMTP发送者的邮件服务器发送邮件到接收者的邮件服务器邮件服务器邮件服务器

SMTP有两个方面:

一是客户端，它在发件人的邮件服务器上执行;

二是服务器端，在收件人的邮件服务器上执行。

  • SMTP的客户端和服务器端都运行在每个邮件服务器上。
  • 当邮件服务器向其他邮件服务器发送邮件时，它充当SMTP客户端。
  • 当邮件服务器从其他邮件服务器接收邮件时，它充当SMTP服务器的角色。

场景:Alice向Bob发送消息

  1. Alice使用用户(UA)代理发送消息到bob@someschool.edu

  2. Alice的UA发送消息到她的邮件服务器

  3. SMTP客户端打开与Bob邮件服务器的TCP连接

  4. SMTP客户端通过TCP连接发送Alice的消息

  5. Bob的邮件服务器将消息放入Bob的邮箱

  6. Bob调用他的用户代理读取消息

DNS服务主机名到IP地址转换

www.someschool.edu的IP地址。是按如下方式完成的。

  1. 同一用户机器运行DNS应用程序的客户端。
  2. 浏览器从URLwww.someschool.edu提取主机名，并将主机名传递给DNS应用程序的客户端。
  3. DNS客户端向DNS服务器发送一个包含主机名的查询。
  4. DNS客户机最终收到一个回复，其中包括主机名的IP地址。
  5. 一旦浏览器从DNS接收到IP地址，它就可以发起一个TCP连接到位于该IP地址的端口80的HTTP服务器进程
